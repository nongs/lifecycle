# [화면 기획] LifeCycle — MVP 화면·플로우

> 관련 문서: `LifeCycle_Project_Plan.md`, `LifeCycle_Development_Plan.md`

> **확정 방향 (2025):** 대시보드 = 메인 전용(가벼움) / 항목·카테고리·로그 CRUD = 항목 관리 페이지

## 1. 화면 구성 요약 (IA)

MVP는 **2개의 메인 페이지** + **레이어 팝업(풀모달·모달)** 로 구성한다.

| 구분 | 이름 | URL (안) | 형태 | MVP |
| :--- | :--- | :--- | :--- | :---: |
| **S-01** | 대시보드 (메인) | `/` | 페이지 | ✅ |
| **S-02** | 항목 관리 | `/items` | 페이지 | ✅ |
| **L-01** | 수행 기록 추가·수정 | — | 모달 | ✅ |
| **L-02** | 항목 추가·수정 | — | 풀모달 | ✅ |
| **L-03** | 항목 상세·로그 목록 | — | 풀모달 | ✅ |
| **L-04** | 카테고리 편집 | — | 풀모달 | ✅ |
| **L-05** | 카테고리 추가·수정 | — | 모달 (L-04 내) | ✅ |
| **L-06** | 확인 (삭제·아카이브) | — | 모달 | ✅ |
| **S-03** | 통계 | `/stats` | 페이지 | ✅ |
| **S-04** | 마이페이지 | `/my` | 페이지 | 고도화 (로그인 전: 설정만) |

### 1.1 역할 분리 원칙

| | 대시보드 (S-01) | 항목 관리 (S-02) |
| :--- | :--- | :--- |
| **목적** | “지금 무엇을 해야 하는가” | “항목·카테고리·기록을 어떻게 관리할 것인가” |
| **항목 CRUD** | ❌ | ✅ 생성·수정·아카이브·삭제 |
| **카테고리** | 카드 **라벨만** 표시 | 필터 chip + **카테고리 편집** (L-04) |
| **로그** | **완료(기록) 추가만** (L-01) | 추가·조회·수정·삭제 (L-01, L-03) |
| **정렬** | 고정: **지연 → 임박 → 여유** | 기본: **생성일**, 사용자 정렬 옵션 추가 예정 |
| **목록 UI** | **1단 목록 고정** | 1단 목록 (관리용) |

대시보드에 항목·카테고리 생성/삭제를 두지 않아 **메인 화면 부담을 줄이고**, 요약 패널·주기 기반 정렬에 집중한다.

### 1.2 글로벌 네비게이션 (MVP)

```
[ 대시보드 ]  [ 관리 ]  [ 통계 ]
     /          /items    /stats
```

* 하단 탭바 또는 상단 탭 — 구현 시 택일.
* 통계·마이페이지 탭은 추후 슬롯만 확보.

## 2. S-01 대시보드 (`/`) — 메인 전용

### 2.1 목적

활성 항목 중 **처리가 시급한 것부터** 보여 주고, **이번 달 아직 처리하지 않은 일정 수**를 요약한다.  
기존 항목에 대한 **수행 로그 기록(완료)** 만 수행한다.

### 2.2 레이아웃 (1단 목록 고정)

```
┌─────────────────────────────────────┐
│  Header: LifeCycle                  │
├─────────────────────────────────────┤
│  ┌─ 요약 패널 ─────────────────────┐ │
│  │  이번 달 일정                    │ │
│  │  N건  (아직 처리하지 않은 항목)   │ │
│  └─────────────────────────────────┘ │
├─────────────────────────────────────┤
│  ┌─ 항목 행 (1열) ─────────────────┐ │
│  │ [지연] 이름  [카테고리 라벨]     │ │
│  │ 신호등 · 진행바 · 경과/잔여       │ │
│  │              [완료]             │ │
│  └─────────────────────────────────┘ │
│  … (지연 → 임박 → 여유 순)           │
└─────────────────────────────────────┘
│  [ 대시보드 ]  [ 항목 관리 ]          │  ← 글로벌 nav
└─────────────────────────────────────┘
```

* **그리드 2열 없음** — 모든 뷰포트에서 1단 목록.
* **카테고리 필터·카테고리 편집·항목 추가 FAB 없음.**

### 2.3 요약 패널 — “이번 달 일정”

| 항목 | 내용 |
| :--- | :--- |
| 표시 | 예: **이번 달 일정 · N건** |
| 의미 | **이번 달 안에 처리가 필요한데, 아직 처리(수행 로그)하지 않은** 활성 항목 개수 |
| 집계 규칙 (안) | 아래 §2.6. 구현 전 비즈니스 규칙 한 번 더 확정 가능 |

패널 탭 시: 해당 항목만 필터링해 목록에 표시할지, 또는 스크롤만 이동할지 — **MVP는 숫자 표시 + 목록이 이미 동일 정렬**이면 패널은 정보용만으로도 충분.

### 2.4 목록 정렬 (고정, 사용자 변경 불가)

| 순위 | 구간 | 조건 (안) |
| :---: | :--- | :--- |
| 1 | **지연** | 경과일 ≥ 목표 주기 (적색) |
| 2 | **임박** | 경과일 ≥ 주기의 80% & 미만 100% (황색) |
| 3 | **여유** | 그 외 (녹색) |

* 동일 구간 내: 경과일 **내림차순**(지연·임박), 여유는 잔여일 **오름차순** 등 — 구현 시 tie-break 명시.
* **로그 없음** 항목: **임박 이상 우선**으로 상단 배치(처리 유도). empty-state UI는 §2.5.

### 2.5 항목 행 UI

| 요소 | 내용 |
| :--- | :--- |
| 이름 | 항목명 |
| 카테고리 | **라벨(chip)만** — 필터·편집 없음 |
| 상태 | 신호등 + 텍스트(지연/임박/여유) + 진행바 |
| 수치 | 경과 N일 · 잔여 M일 (또는 “N일 지남”) |
| CTA | **[완료]** → L-01 (수행일·메모·비용) |

**카드 상태별**

| 상태 | 표시 |
| :--- | :--- |
| 로그 없음 | 중립 배지 + “기록 없음”. 진행바 생략 또는 placeholder. [완료] → L-01 |
| 오늘 수행 / 주기 0일 | “오늘 수행함” 배지·아이콘 |
| 지연 | 적색 강조, “N일 지남” |
| 일반 | 신호등 + 진행바 + 경과/잔여 |

### 2.6 “이번 달 미처리” 집계 규칙 (안)

> 개발 착수 시 `utils/dashboardSummary.ts` 등으로 일원화.

* 대상: `status === 'active'` 인 항목.
* **미처리:** 이번 달(달력 월, 0시 기준)에 **해당 항목의 수행 로그가 없음**.
* **처리 필요:** 다음 중 하나  
  - 이미 **지연** 상태이거나  
  - **다음 예정일**(최신 로그 + `targetCycleDays`)이 **이번 달 말일 이내**이거나  
  - **로그가 한 번도 없음** (첫 기록 유도)

### 2.7 대시보드에서 허용하는 모달

| ID | 이름 | 진입 |
| :--- | :--- | :--- |
| **L-01** | 수행 기록 추가·수정 | 행 [완료] |

* **L-03 항목 상세·로그 목록은 대시보드에서 열지 않음** — 로그 이력·수정·삭제는 항목 관리(S-02)에서만.

### 2.8 Empty State

| 상황 | UI |
| :--- | :--- |
| 활성 항목 0개 | “관리할 항목이 없습니다” + **[항목 관리로 이동]** → `/items` |
| 목록은 있으나 모두 여유 | 요약 N=0, 목록은 여전히 표시 가능 |

## 3. S-02 항목 관리 (`/items`)

### 3.1 목적

항목·카테고리·로그에 대한 **전체 관리 기능**. 생성·삭제·정렬·필터·상세·아카이브를 이 화면에서만 수행한다.

### 3.2 레이아웃

```
┌─────────────────────────────────────┐
│  Header: 항목 관리        [＋ 항목]  │
├─────────────────────────────────────┤
│  [전체] [카테고리1] [카테고리2] …     │  ← 카테고리 필터 chip
│                      [카테고리 편집] │  → L-04 풀모달
├─────────────────────────────────────┤
│  정렬: [생성일 ▼]  (추가 옵션 예정)   │
├─────────────────────────────────────┤
│  ┌─ 항목 행 ──────────────────────┐ │
│  │ 이름 · 카테고리 · 주기 · 상태    │ │
│  │        [상세]  [⋮ 메뉴]         │ │
│  └─────────────────────────────────┘ │
│  …                                  │
├─────────────────────────────────────┤
│  (선택) 아카이브 섹션 / 탭           │
└─────────────────────────────────────┘
```

### 3.3 카테고리 필터

| chip | 동작 |
| :--- | :--- |
| **전체** | 활성 항목 모두 |
| **카테고리 N** | `categoryId` 일치 항목만 |

* chip 목록 = 사용자 `Category` 목록 (시드 포함).
* **카테고리 편집** 버튼 → **L-04 풀모달** (수정·삭제·**순서 변경**).

### 3.4 정렬

| 구분 | 규칙 |
| :--- | :--- |
| **기본값** | 항목 **생성일(`createdAt`)** 기준 (최신 또는 오래된 순 — MVP에서 하나로 고정, UI에 표시) |
| **추가 예정** | 이름, 카테고리, 주기, 최근 수행일 등 — 정렬 UI는 플레이스홀더·확장 포인트로 설계 |

대시보드 정렬(지연→임박→여유)과 **독립**이다.

### 3.5 항목 행 액션

| 액션 | 결과 |
| :--- | :--- |
| **상세** | **L-03 풀모달** — 항목 정보 + 로그 타임라인 |
| **＋ 항목** / 행 수정 | **L-02** 항목 추가·수정 |
| **⋮ 메뉴** | 아카이브, 영구 삭제(L-06), (선택) 빠른 기록 추가 → L-01 |

### 3.6 아카이브

* 목록 하단 **섹션** 또는 **탭「아카이브」**: `status === 'archived'` 항목.
* **복구**, **영구 삭제** (L-06).

## 9. 확정 사항 · 후속 산출물

### 확정됨

- [x] 대시보드 = 단일 메인, 로그 **추가(완료)** 만
- [x] 항목·카테고리·로그 CRUD·상세 = **항목 관리 페이지**
- [x] 상세·로그 목록 = **풀모달 (L-03)**, 항목 관리에서만
- [x] 대시보드 1단 목록, 카테고리는 **라벨만**
- [x] 대시보드 정렬: **지연 → 임박 → 여유**
- [x] 항목 관리 정렬: 기본 **생성일**, 옵션은 추후
- [x] 비용: 로그당 **선택**, **체크박스로 필드 노출**
