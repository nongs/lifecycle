# [기획안] 일상 사이클 관리 서비스 (가칭: LifeCycle)

## 1. 서비스 개요
* **컨셉:** 자동차 소모품 관리 시스템을 일상생활 전반으로 확장한 '역방향 일정 관리' 서비스.
* **핵심 가치:** "미래의 할 일"이 아닌 **"과거의 마지막 수행일"**로부터 경과된 시간을 추적하여 적절한 루틴 유지를 도움.
* **대상:** 미용, 가전 필터 교체, 쇼핑 주기, 자동차 정비 등 반복되는 일상을 체계적으로 관리하고 싶은 사용자.

## 2. 주요 기능 (Functional Requirements)

### ① 항목별 주기 설정 및 대시보드
* **항목 생성:** 관리할 대상(예: 이발, 주유, 옷 구입, 칫솔 교체 등) 등록.
* **권장 주기 설정:** 수행해야 할 목표 주기 설정(예: 이발 - 4주, 엔진오일 - 6개월).
* **D-Day 및 경과일 표시:** * **경과 시간(Elapsed Time):** "마지막으로 수행한 지 며칠 지났는가" 표시.
    * **잔여 시간(Remaining Time):** "다음 예정일까지 며칠 남았는가" 표시.

### ② 히스토리 로그 (Log Management)
* **원클릭 기록:** 오늘 수행했을 경우 버튼 한 번으로 날짜 업데이트.
* **과거 기록 수정:** 실제 수행일과 기록일이 다를 경우 날짜 소급 적용.
* **메모 기능:** 간단한 텍스트 기록 (예: 주유 시 리터당 가격, 옷 구매 시 브랜드 등).

### ③ 상태 시각화 (Status Visualization)
* **신호등 시스템:** * **안전(녹색):** 수행한 지 얼마 안 된 상태.
    * **주의(황색):** 설정한 주기의 80% 이상 경과.
    * **지연(적색):** 설정한 주기를 초과함.
* **진행바(Progress Bar):** 현재 주기의 어느 지점에 있는지 시각적으로 표현.

### ④ 통계 및 인사이트
* **평균 주기 계산:** 사용자의 실제 기록 데이터를 바탕으로 평균 수행 주기 산출.
* **비용 추적:** 항목별 지출 금액 입력 시 월별/항목별 총지출 통계 제공.

## 3. 카테고리 구성 예시
* **그루밍:** 미용실, 네일, 스케일링, 화장품 개봉.
* **모빌리티:** 주유, 엔진오일, 와이퍼, 타이어 로테이션.
* **리빙:** 정수기 필터, 에어컨 청소, 침구 세탁, 칫솔 교체.
* **소비:** 의류 구입, 운동화 구입, 영양제 구매.

## 4. 시스템 구조 (Technical Architecture)

### [데이터베이스 설계(안)]
| 테이블 | 필드 구성 |
| :--- | :--- |
| **Category** | 카테고리명, 아이콘, 사용자ID |
| **Items** | 항목명, 목표 주기(Days), 카테고리ID, 알림 설정 여부 |
| **Logs** | 수행 날짜, 비용, 메모, ItemID |

### [기술 스택 추천]
* **Frontend:** React.js 또는 Next.js (반응형 웹 및 PWA 고려)
* **Backend:** Node.js (Express) 또는 Supabase / Firebase (BaaS)
* **Database:** PostgreSQL (Supabase) 또는 NoSQL (Firebase)

## 5. 단계별 개발 로드맵
1.  **1단계 (MVP):** 항목 등록, 날짜 기록, 경과일 표시 기능 구현.
2.  **2단계 (통계):** 비용 입력 및 항목별 평균 주기 통계 추가.
3.  **3단계 (알림/앱):** 브라우저 푸시 알림 구현 및 PWA(앱처럼 설치) 적용.
