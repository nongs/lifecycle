# LifeCycle — Cloud ↔ Local 데이터 동기화 (B-2)

## 원칙

| 상태 | 진실(source of truth) | 저장소 |
|------|----------------------|--------|
| 미로그인 | 이 기기 local | `localStorage` |
| 로그인 중 | 계정(클라우드) | Supabase |
| 로그아웃 후 | 스냅샷 복사본 | `localStorage` (동기화 중단, 읽기용 게스트) |

**로컬 데이터 클리어는 하지 않는다.** 로그아웃 시 클라우드 최신 상태를 로컬에 덮어쓴 뒤 세션만 종료한다.

---

## 로그인 직후

```
로그인 성공
  ├─ 클라우드에 항목/로그 있음 → 클라우드만 표시 (다른 기기와 동일)
  ├─ 클라우드 비어 있음 + 로컬에 항목/로그 있음 → 마이그레이션 제안 (1회 업로드)
  └─ 둘 다 비어 있음 → 클라우드 시드 카테고리 (기존 seed.ts와 동일)
```

- 「클라우드 비어 있음」= `management_items` 0건 **且** `activity_logs` 0건 (카테고리만 있는 시드 상태는 비어 있음으로 간주)
- 마이그레이션 시 기존 클라우드 행(시드 카테고리 포함) 삭제 후 로컬 전체를 업로드, ID는 새 uuid로 매핑

---

## 로그아웃

1. 로그인 중이면 `snapshotCloudToLocal()` — 클라우드 전체 → localStorage 덮어쓰기
2. `signOut()` — 세션만 종료
3. `dataMode` → `local`, 방금 스냅샷으로 앱 계속 사용 가능

다른 기기에서 수정한 내용은 로그아웃 상태에 반영되지 않음 (의도된 동작).

|------|------|
| `src/lib/data/cloudLocalSync.ts` | 읽기/쓰기, 스냅샷, 마이그레이션, 로그인 직후 판단 |

## 데이터 보내기·가져오기 (`/settings`)

- **보내기**: JSON 파일 저장 또는 클립보드 복사 (`datasetIO.ts`, `version: 1`)
- **가져오기**: 파일 선택 또는 JSON 붙여넣기 → 현재 데이터 소스(로컬/클라우드) **전체 덮어쓰기**
- 테스트·백업·기기 간 수동 이전용

## B-2 이후 (선택)

- 로그인 중 주기적 cloud → local 캐시 (오프라인/PWA)
- 충돌 UI (로컬·클라우드 둘 다 데이터 있을 때)
